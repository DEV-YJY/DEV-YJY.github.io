Class and IDs in CSS

CSS stands for Cascading Style Sheets. CSS is responsible for beautifying the websites. Without CSS you will find internet surfing extremely boring and difficult.


In CSS, styles are applied by targeting a specific element or range of elements with different classes and IDs. Then, targeted elements get styled!

As the name suggests, CSS classes and ID pose some differences that you need to be careful when utilising them. The difference is that the CSS class applies a style to multiple elements whereas ID can only apply styles to one unique element. This means that once attached, the ID will not be able to be reused within the same webpage.

IDs also have a higher specificity than classes meaning, styles that belong to an ID can overwrite the ones that belong to classes.

There are some more differences and details that can be overviewed to discuss the practicability of using CSS classes and IDs but they will not be covered in this post.


I personally think styles should be easy to override and extended to make CSS pleasant to work with and maintainable. Also, if I was working with my co-worker on a project, I would prefer to work only with the class attribute to define styles rather than seeing class and ID in a combination.

Thus, based on what we have briefly covered.. although there are no rules that stop you from using ID over a class, I believe it is a better practice to keep the usage of IDs to the minimum and only apply it where necessary. By doing so it will be able to minimise any potential confusion :)
